# 有效提示词写法

## 场景

与 agent 交互时，如何写出更有效的提示词（prompt）。

## 技巧

### 1. 结构化提示词

使用清晰的结构组织提示词：
- **角色**：告诉 agent 扮演什么角色
- **任务**：明确描述需要完成的任务
- **上下文**：提供必要的背景信息
- **约束**：说明限制条件
- **输出格式**：指定期望的输出格式

### 2. 具体化而非模糊

❌ 模糊：`帮我写一个函数`
✅ 具体：`用 Python 写一个函数，接受一个整数列表，返回所有偶数之和`

### 3. 示例驱动

提供具体的输入输出示例：
```
输入：[1, 2, 3, 4, 5]
期望输出：6 (2+4)
```

### 4. 分步骤请求

对于复杂任务，分解为多个步骤：
1. 先让 agent 理解问题
2. 提出解决方案
3. 实现具体代码
4. 验证和测试

## 示例

```
你是一位经验丰富的 Python 开发者。

任务：实现一个数据验证函数
要求：
- 验证用户输入的邮箱地址格式
- 返回布尔值
- 对于非法输入（如 None）要有适当处理

示例：
- is_valid_email("user@example.com") -> True
- is_valid_email("invalid-email") -> False
- is_valid_email(None) -> False
```

## 注意事项

- 提示词越清晰，结果越准确
- 如果结果不满意，尝试重新表述或添加更多约束
- 保存有效的提示词模板以便复用
