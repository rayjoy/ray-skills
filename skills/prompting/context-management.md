# 上下文管理

## 场景

在长对话中与 agent 交互时，如何有效管理上下文。

## 技巧

### 1. 及时总结

在长对话中，定期请求 agent 总结已完成的工作：
```
请总结一下我们到目前为止做了什么，以及当前的状态。
```

### 2. 明确引用之前的内容

当继续之前的工作时，明确引用相关内容：
```
继续我们之前讨论的用户认证模块，现在我们需要...
```

### 3. 重置上下文

当话题转换时，明确告知 agent：
```
我们现在切换到另一个话题：[新话题]
```

### 4. 分段处理大任务

对于大型任务，分成多个独立的对话：
- 每个对话专注于一个子任务
- 在每个对话开始时提供必要的上下文
- 使用文件或文档记录各部分的结果

## 示例

开始新任务时的上下文设置：
```
背景：我正在开发一个电商平台的后端服务
技术栈：Node.js, Express, PostgreSQL
当前任务：实现商品搜索功能
约束：需要支持模糊搜索和过滤

请帮我设计这个功能的实现方案。
```

## 注意事项

- 上下文窗口有限制，避免在一个对话中处理过多内容
- 重要的决定和代码要单独保存，不要只依赖对话历史
- 对话结束前总结关键点，方便下次继续
