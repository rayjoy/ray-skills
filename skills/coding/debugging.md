# 调试技巧

## 场景

使用 agent 辅助排查和解决 bug 时的技巧。

## 技巧

### 1. 提供完整的错误信息

- 完整的错误堆栈（stack trace）
- 错误发生的环境信息（操作系统、语言版本、框架版本）
- 触发错误的具体步骤

### 2. 二分法定位问题

让 agent 帮助使用二分法缩小问题范围：
- 先确认问题在哪个模块
- 再缩小到具体的函数
- 最后定位到具体的代码行

### 3. 日志分析

提供相关日志，让 agent 分析：
- 异常模式
- 时间关联性
- 关键变量的状态变化

## 示例

```
我遇到了以下错误：

错误信息：[错误内容]
错误堆栈：[堆栈信息]
触发条件：[操作步骤]
环境：[环境信息]

相关代码：
[代码内容]

请帮我分析可能的原因和解决方案。
```

## 注意事项

- 确保提供的代码片段包含足够的上下文
- 如果是间歇性 bug，需要说明触发频率和条件
- 说明已经尝试过的解决方案，避免 agent 建议重复的方法
