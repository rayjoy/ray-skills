# 重构技巧

## 场景

使用 agent 辅助进行代码重构时的技巧。

## 技巧

### 1. 明确重构目标

在请求重构前，明确以下目标：
- 提高可读性
- 提升性能
- 降低复杂度
- 消除重复代码（DRY 原则）
- 提高可测试性

### 2. 增量重构

- 每次只重构一个小的部分
- 确保每次重构后测试仍然通过
- 逐步积累改进

### 3. 保持功能不变

请求 agent 重构时明确说明：
- 外部接口不能改变
- 行为必须与原来一致
- 需要保留的注释和文档

## 示例

```
请重构以下代码，目标是：
1. 提高可读性，使用更清晰的变量名
2. 消除重复逻辑
3. 保持现有的功能和接口不变

[代码内容]
```

## 注意事项

- 重构前确保有足够的测试覆盖
- 大型重构分多个步骤进行
- 每次重构后运行所有测试确保没有引入 bug
