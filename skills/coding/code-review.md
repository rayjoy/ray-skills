# 代码审查技巧

## 场景

使用 agent 进行代码审查时的技巧。

## 技巧

### 1. 提供充分上下文

在请求代码审查时，提供以下上下文：
- 代码的目的和功能
- 相关的业务逻辑背景
- 已知的约束条件

### 2. 分层次审查

- 先请 agent 检查整体架构和设计问题
- 再检查具体的逻辑和算法
- 最后检查代码风格和规范

### 3. 针对性提问

- "这段代码有没有安全漏洞？"
- "这个函数的时间复杂度是多少，有优化空间吗？"
- "这段代码是否遵循了 SOLID 原则？"

## 示例

```
请审查以下代码，重点关注：
1. 潜在的 bug
2. 性能问题
3. 安全隐患

[代码内容]
```

## 注意事项

- 不要一次性提交太多代码，分批审查效果更好
- 对 agent 的建议要保持批判性思维，不要盲目接受
- 重要的修改需要自己再次确认
