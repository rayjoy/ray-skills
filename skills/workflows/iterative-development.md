# 迭代开发流程

## 场景

使用 agent 辅助迭代开发时的最佳实践。

## 技巧

### 1. 最小可行版本优先

先实现最简单的可工作版本，再逐步完善：
1. 实现核心功能（MVP）
2. 添加错误处理
3. 优化性能
4. 完善文档和测试

### 2. 快速反馈循环

建立快速的反馈机制：
- 每次修改后立即测试
- 使用 agent 帮助分析测试结果
- 根据反馈快速调整方向

### 3. 代码审查集成

在每个迭代结束时：
- 让 agent 审查本次迭代的代码
- 确保代码质量不会随迭代下降
- 及时偿还技术债务

## 示例

迭代开发流程：

```
第一轮迭代：
目标：实现基本的 CRUD 操作
[实现 → 测试 → 审查]

第二轮迭代：
目标：添加输入验证和错误处理
[实现 → 测试 → 审查]

第三轮迭代：
目标：添加缓存优化性能
[实现 → 测试 → 审查]
```

## 与 Agent 协作的迭代流程

```
1. 描述当前迭代目标
2. 让 agent 提出实现方案
3. 讨论和调整方案
4. 让 agent 实现代码
5. 运行测试验证
6. 请 agent 进行代码审查
7. 修改并确认
8. 开始下一轮迭代
```

## 注意事项

- 每次迭代保持小而聚焦，避免范围蔓延
- 保持良好的文档，记录每次迭代的决策
- 定期回顾和调整开发策略
