# 任务分解

## 场景

使用 agent 处理复杂任务时，如何有效分解任务。

## 技巧

### 1. 自顶向下分解

让 agent 帮助将大任务分解为小任务：
```
我需要实现一个[功能描述]。
请帮我将这个任务分解为独立的子任务，
每个子任务应该可以独立完成和验证。
```

### 2. 依赖分析

分解后，明确任务间的依赖关系：
- 哪些任务可以并行
- 哪些任务必须按顺序执行
- 关键路径是什么

### 3. 定义完成标准

每个子任务都要有明确的完成标准（Definition of Done）：
- 功能要求
- 测试要求
- 文档要求

## 示例

```
我需要实现一个用户管理系统，包含：
- 用户注册和登录
- 权限管理
- 用户信息管理

请帮我：
1. 将这个系统分解为具体的开发任务
2. 估算每个任务的复杂度（简单/中等/复杂）
3. 确定任务的推荐执行顺序
4. 指出可以并行进行的任务
```

## 注意事项

- 任务粒度要适中：太小会过于繁琐，太大则难以跟踪
- 每个任务完成后及时更新状态
- 遇到新需求时重新评估任务分解
